# 01. Context — 왜 어드민을 만드는가

---

## 1. 프로젝트 배경

### Novel Agent 란
한국 웹소설 작가를 대상으로 한 **AI 기반 흥행 분석 SaaS**.
- 작가가 본인 원고(회차별)를 업로드하면 AI가 분석
- 카카오페이지/문피아 등 플랫폼 트렌드 기반 흥행 가능성 리포트 제공
- 개별 회차 분석 + 다회차 통합 분석 2가지
- 2026년 4월 현재 베타 운영 중 (novelagent.kr)

### 운영 구조
- **1인 운영** (LEE, 前 웹소설 플랫폼 PM)
- **비개발자 배경** — AI 코딩 도구(Cursor) + Claude Code 활용
- 현재 베타 무료 → 패들 결제 연동 후 유료 전환 예정 (9월 사업자 등록 후)

---

## 2. 해결해야 할 문제

### 현재 운영상 문제
LEE 가 서비스 운영 중 **반복적으로 SQL / Supabase 콘솔 수작업** 중:

| 상황 | 현재 방식 | 빈도 | 리스크 |
|-----|---------|-----|------|
| 유저 문의 들어옴 | Supabase 들어가서 users 테이블 뒤져 이메일 확인 | 주 2~5회 | 실수 위험 낮음 |
| NAT 잔량 조정 필요 | SQL 수기로 `UPDATE users SET nat_balance = ...` | 주 1~2회 | **높음** (오입력) |
| 유저 문의 답장 후 상태 표시 | 현재 답장 상태 기록 안 됨 | 주 2~5회 | 중복 답장 / 누락 |
| 탈퇴 이유 분석 | SQL 로 withdrawal_logs 조회 | 주 1회 | 정성적 분석 불가 |

### 핵심 고통 포인트
1. **NAT 수동 조정 시 오타 위험** — 잘못 입력하면 유저 자산 증발
2. **문의 답장 추적 안 됨** — 같은 유저에게 두 번 답장 보낸 사례 발생
3. **데이터 맥락 없음** — SQL 로 숫자만 나오고, 해당 유저의 다른 정보와 연결 안 됨

---

## 3. 어드민의 목적

### MUST HAVE (이번 MVP)
1. **유저 조회** — 이메일/닉네임 검색 → 상세 정보 (가입일, NAT 잔량, 작품 수, 분석 횟수)
2. **NAT 수동 조정 + 로그** — 버튼 UI 로 충전/차감, 모든 조정 기록 남김
3. **문의 목록 + 상태** — 누구에게 답장했는지, 안 했는지 명확히
4. **탈퇴 로그** — 탈퇴한 유저의 이유 + 패턴 관찰

### NICE TO HAVE (다음 스프린트)
- 분석 job 재시도 UI
- 작품/회차 검색 및 디버깅
- 공지사항 CMS

### NOT NOW (범위 외)
- 대시보드 통계 (DAU/MAU/차트)
- A/B 테스트 도구
- 이메일 대량 발송
- Paddle 결제 관련 UI (별도 스프린트)

## 4. 사용자 페르소나

### 현재: LEE (1명)
- 前 웹소설 플랫폼 PM (5년)
- 비개발자 배경
- Mac 작업 환경
- **데이터 신뢰도 중시** — 실수 방지 UI 선호
- **빠른 작업 흐름 선호** — 클릭 3번 이내로 주요 작업 완료

### 미래: CS 담당자 (예정, 6개월 내 가능성)
- 운영 보조 직원 (가상)
- 기술적 이해 낮음 → UI 만으로 대부분 처리 가능해야
- 권한 제한 필요 (예: NAT 차감 불가, 충전만 가능 등)
  - → 이번 MVP 는 권한 세분화 **안 함** (LEE 전용)
  - → 단, 나중에 확장 가능하게 `users.role` 필드 구조는 잡아둘 것

## 5. 제약 조건

### 기술
- Next.js 16 + Supabase 기존 스택 유지
- Claude Code 가 자율 구축 → **표준 Next.js 패턴** 선호
- Vercel 서버리스 환경 (단일 요청 300초 제한)

### 시간
- **MVP 3~5일 내 완성** 목표
- 이후 소비자 사이트 (사이드 패널) 작업과 병행 진행

### 품질
- **작동하는 것 우선 > 예쁜 것**
- 스크린샷 남길 수 있는 수준 (포트폴리오 의식)
- 에러 처리 필수 (운영 데이터 다루므로)

### 보안
- `/admin` 경로는 **Middleware 에서 차단**
- 비인가 접근 시 404 또는 /login 리다이렉트
- 관리자 행동 로그 남기기 (NAT 조정 등은 누가 언제 얼마만큼)

---

## 6. 성공 기준

### 정성
- LEE 가 SQL 치는 빈도 **80% 감소**
- 문의 답장 중복/누락 **0회**
- NAT 수동 조정 실수 **0회** (모달 확인 단계 필수)

### 정량
- 유저 검색 → 상세 진입 **3초 이내**
- NAT 조정 후 잔량 반영 **1초 이내**
- 문의 목록 로딩 **2초 이내** (100건 기준)
