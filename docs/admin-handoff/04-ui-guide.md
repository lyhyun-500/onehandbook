# 04. UI Guide — 디자인 원칙

---

## 🎨 기본 원칙

### 참고 UI: Supabase Dashboard
- 이유: 어드민 UI 의 정석, Claude Code 가 참고할 표준 존재
- 핵심: **데이터 밀도 + 기능 우선 + 밝은 테마**

### 반대로, 참고 안 할 것
- 소비자 사이트의 다크 테마 → **어드민은 밝은 테마**로 구분
- Notion 스타일의 화려한 일러스트 → 어드민은 실용 우선
- 풍부한 애니메이션 → 필요 최소한만

## 🌕 테마

### 색상 시스템
어드민은 **밝은 테마 (Light Mode)** 고정.

```css
/* 추천 팔레트 (Tailwind 표준 색 활용) */

/* 배경 */
--admin-bg-page: #ffffff        /* 페이지 배경 (white) */
--admin-bg-surface: #f9fafb     /* 카드/테이블 배경 (gray-50) */
--admin-bg-hover: #f3f4f6       /* hover 배경 (gray-100) */

/* 테두리 */
--admin-border: #e5e7eb         /* gray-200 */
--admin-border-strong: #d1d5db  /* gray-300 */

/* 텍스트 */
--admin-text-primary: #111827   /* gray-900 */
--admin-text-secondary: #4b5563 /* gray-600 */
--admin-text-muted: #9ca3af     /* gray-400 */

/* 액센트 */
--admin-accent: #2563eb         /* blue-600 (주요 버튼) */
--admin-accent-hover: #1d4ed8   /* blue-700 */

/* 상태 */
--admin-success: #059669        /* green-600 */
--admin-warning: #d97706        /* amber-600 */
--admin-danger: #dc2626         /* red-600 */
--admin-info: #0891b2           /* cyan-600 */
```

**구현 방식**:
- Tailwind v4 `@theme` 블록에 추가 (또는 기본 Tailwind 색 사용)
- 소비자 사이트의 `--color-sidepanel-*` 토큰과 **완전 분리**
- 네임스페이스: `--color-admin-*`

## 📐 레이아웃

### 전체 구조
┌─────────────────────────────────────────────────┐
│  [사이드바]    [메인 컨텐츠]                      │
│               ┌───────────────────────┐          │
│  ☰ 홈          │ 페이지 제목           │          │
│    유저        │ [검색/필터]            │          │
│    문의        │                       │          │
│    탈퇴        │ [데이터 테이블/카드]   │          │
│               │                       │          │
│  ─── 구분선 ─── │                      │          │
│    설정        │                       │          │
│    로그아웃    │                       │          │
│               └───────────────────────┘          │
└─────────────────────────────────────────────────┘

### 사이드바 (좌측 고정)
- 너비: 240px
- 배경: white
- 오른쪽 테두리: 1px solid gray-200
- 메뉴 항목:
  - 홈 (`/admin`) — 홈 아이콘
  - 유저 관리 (`/admin/users`) — 사람 아이콘
  - 문의 관리 (`/admin/inquiries`) — 말풍선 아이콘
  - 탈퇴 로그 (`/admin/withdrawals`) — 나가는 문 아이콘
  - 구분선
  - 로그아웃 버튼

- **현재 페이지 하이라이트**: 좌측 3px 파란 바 + 배경 gray-100
- **아이콘 라이브러리**: `lucide-react` (소비자 사이트와 동일하면 OK)

### 메인 컨텐츠 영역
- 좌측 margin 240px (사이드바 너비)
- 최대 너비 1400px
- padding: 32px

### 페이지 헤더
- 제목 (h1, 24px, weight 600)
- 부제 (선택, 14px, gray-600)
- 우측: 주요 액션 버튼 (있다면)

## 🧩 공용 컴포넌트

### 1. 카드 (Card)
┌──────────────────────┐
│ 제목                  │
│ ────────             │
│ 내용                  │
│                      │
└──────────────────────┘
- 배경: white
- 테두리: 1px solid gray-200
- border-radius: 8px
- padding: 24px
- 그림자: 없거나 매우 약하게 (`shadow-sm`)

### 2. 통계 카드 (StatCard)
대시보드 홈에서 사용.
┌──────────────────┐
│ 유저 총 수        │  ← label (14px, gray-600)
│                  │
│ 1,234            │  ← value (32px, weight 700)
│                  │
│ ↑ 오늘 +12명      │  ← delta (12px, green-600)
└──────────────────┘

- 배경: white
- 테두리: 1px solid gray-200
- padding: 24px
- 변화값 (delta) 은 있으면 표시

### 3. 테이블 (DataTable)
┌──────────┬──────────┬──────────┬──────────┐
│ 닉네임    │ 이메일    │ 가입일    │ 액션      │
├──────────┼──────────┼──────────┼──────────┤
│ 이묵검    │ lee@...  │ 04-21    │ [상세]    │
│ 주현정    │ joo@...  │ 04-20    │ [상세]    │
└──────────┴──────────┴──────────┴──────────┘

- 헤더: 배경 gray-50, 텍스트 weight 600, 14px
- 행: 높이 48~56px, hover 시 배경 gray-50
- 행 구분선: 1px solid gray-200
- 텍스트: 14px
- 액션 버튼: 작은 outlined 버튼 (파란 테두리)

### 4. 검색/필터 바
┌──────────────────────────────────────────┐
│ 🔍 [검색어 입력                    ]      │
│ [경로▼] [상태▼] [정렬▼] [검색] [초기화]    │
└──────────────────────────────────────────┘

- 배경: white, 카드 안에 배치
- 입력창: 36px 높이, gray-300 테두리
- 셀렉트: 36px 높이
- 버튼:
  - 주요 (검색): 파란 배경 (blue-600), 흰 글자
  - 보조 (초기화): 테두리만, gray-700 글자

### 5. 뱃지 (Badge)
상태 표시용.
활성:    [활성]       ← green-100 배경 + green-700 글자
탈퇴:    [탈퇴]       ← red-100 배경 + red-700 글자
미답장:  [미답장]     ← amber-100 배경 + amber-700 글자
답장완료: [답장완료]   ← gray-100 배경 + gray-700 글자

- 높이: 22px
- padding: 2px 8px
- border-radius: 4px (또는 full)
- 폰트: 12px weight 500

### 6. 버튼 (Button)

#### Primary
- 배경: blue-600, hover blue-700
- 글자: white
- padding: 8px 16px
- 용도: 주요 액션 (검색, 저장, 조정)

#### Outlined
- 테두리: 1px solid gray-300, hover gray-400
- 글자: gray-700
- 배경: white, hover gray-50
- 용도: 보조 액션 (초기화, 취소)

#### Danger
- 배경: red-600, hover red-700
- 글자: white
- 용도: 파괴적 액션 (차감, 삭제)

### 7. 모달 (Modal)

NAT 조정 확인용.
┌──────────────────────────────────┐
│ 확인                      [X]     │
├──────────────────────────────────┤
│                                  │
│ 정말 이 유저에게 50 NAT 를          │
│ 충전하시겠습니까?                  │
│                                  │
│ 사유: 이벤트 보상                  │
│                                  │
│                                  │
│           [취소]  [확인 (충전)]     │
└──────────────────────────────────┘

- 오버레이: black 40% 투명
- 모달 본체: white, 최대 너비 480px
- padding: 24px
- 버튼 배치: 우측 정렬 ([취소] [주요 액션])
- 주요 액션 버튼이 파괴적이면 Danger 색 사용

### 8. 토스트 (Toast)
작업 결과 피드백.
┌──────────────────────────────┐
│ ✓ 50 NAT 충전 완료              │
└──────────────────────────────┘

- 위치: 화면 우측 상단 또는 하단 (Claude Code 판단)
- 자동 사라짐: 3초
- 색:
  - 성공: green-50 배경, green-700 글자, green-500 좌측 바
  - 에러: red-50 배경, red-700 글자
  - 정보: blue-50 배경

### 9. 드로어 (Drawer)
문의 상세 표시용.
             ┌──────────────────┐
             │ 문의 상세    [X]   │
             ├──────────────────┤
             │                  │
             │ 유저 정보         │
             │ 제목             │
             │ 내용             │
             │                  │
             │ [이메일 복사]     │
             │ [답장 완료 표시]   │
             │                  │
             └──────────────────┘

- 우측에서 슬라이드 인
- 너비: 480px
- 배경: white
- 클릭 외부 or ESC 키로 닫힘

## 📱 반응형

**반응형 불필요**. 데스크탑 전용으로 가정.
- 최소 너비: 1200px
- 브라우저 창 작으면 가로 스크롤 발생 허용

이유:
- LEE 가 주 사용자 (Mac 데스크탑 확정)
- 어드민 페이지는 스마트폰에서 쓰지 않음
- 반응형 투자 시간 = 낭비

## 🎭 톤앤매너

### 카피
- **간결**: "저장" > "변경사항 저장하기"
- **명확**: "충전" / "차감" (애매한 "조정" 보다)
- **한국어**: 모든 UI 문구 한국어
- **존댓말**: 확인 메시지는 존댓말 ("충전하시겠습니까?")
- **명령형**: 버튼은 짧게 ("검색", "저장", "취소")

### 예외 상황 문구
- **빈 상태**: "표시할 내용이 없습니다"
- **로딩 실패**: "데이터를 불러올 수 없습니다. 새로고침해주세요"
- **권한 없음**: 404 페이지로 리다이렉트 (메시지 노출 X)
- **성공**: "[동작] 완료" (예: "충전 완료")
- **실패**: "[동작] 실패: [이유]" (예: "충전 실패: 잔량 부족")

## 🚫 UI 금기 사항

### 하지 말아야 할 것
- ❌ 이모지 남용 (기능 아이콘은 lucide-react)
- ❌ 그라디언트 배경
- ❌ 과도한 애니메이션 (hover 색 변화 정도만)
- ❌ 컬러풀한 일러스트 (빈 상태는 심플 아이콘 + 텍스트)
- ❌ 모달 중첩 (모달 안에 또 모달 X)
- ❌ 10개 이상 선택지의 드롭다운 (검색 가능하게)

### 해야 할 것
- ✅ 일관된 간격 (4px 배수: 4/8/12/16/24/32)
- ✅ 모든 클릭 가능 요소에 cursor pointer
- ✅ 포커스 아웃라인 (keyboard 접근성)
- ✅ 로딩 상태 표시 (스피너 또는 스켈레톤)
- ✅ 에러 메시지 구체적 (무엇이 왜 실패했는지)

## 🧪 작업 시 체크리스트

각 페이지/컴포넌트 완성 시 아래 확인:

- [ ] 밝은 테마 적용됨 (소비자 사이트 다크 색 없음)
- [ ] 사이드바 현재 페이지 하이라이트
- [ ] 테이블에 hover 효과
- [ ] 모든 액션에 로딩 상태 있음
- [ ] 에러 상태 UI 있음
- [ ] 빈 상태 UI 있음
- [ ] 한국어 존댓말 일관성
- [ ] 포커스 키보드 네비 가능
- [ ] 빌드 경고 없음
