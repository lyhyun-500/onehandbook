## 1. Context (배경)

### 문제
Novel Agent 의 핵심 기능 = AI 기반 웹소설 분석. **AI API (Claude) 호출 비용이 변동비** 라서, 무제한 무료 또는 단순 월정액으로는 비즈니스 지속 불가능.

베타 출시 시점에 **과금 모델을 확정** 해야 했음.

### 비용 구조
- AI API 비용 = **글자수에 거의 비례** (입력 토큰 + 출력 토큰)
- 회차당 글자수 편차 큼 (3,000자 ~ 10,000자)
- 일괄 분석 시 N배 누적 (10화 = 단일의 10배)
- 추가 옵션 (장르별 심화, 비교 분석 등) 마다 추가 호출

### 작가의 사용 패턴
- 라이트 유저: 월 1~2회차 분석 (취미 작가)
- 미들 유저: 월 5~10회차 분석 (등단 준비 작가)
- 헤비 유저: 월 20회차 이상 + 일괄 분석 (기성 / 다작 작가)

### 제약
- 1인 운영, AI API 마진 확보 필수
- 베타 단계 = 사용자 확보 우선, 진입 장벽 낮춰야 함
- 한국 웹소설 시장 = 가격 민감
- 결제 시스템 (패들) 도입 전 = 단순 모델 선호

## 2. Decision (결정)

**NAT (Novel Agent Token) 하이브리드 모델 채택.**

### 구조
1. **베타 (현재)**: 무료, 주 3 NAT 자동 충전
2. **스탠다드 월정액**: 9,900원 / 월 → 100 NAT
3. **NAT 추가 구매** (소진 시):
   - 50 NAT = 4,900원
   - 100 NAT = 8,900원
   - 200 NAT = 15,900원
4. **NAT 차등 소모** (글자수 기준):
   - 3,000자 이하 = 1 NAT
   - ~ 6,000자 = 2 NAT
   - ~ 10,000자 = 3 NAT
   - 옵션 추가 시 +1 NAT 씩

### 핵심 설계 원리
- **사용량 기반 변동비를 NAT 라는 추상 단위로 캡슐화** → 사용자는 원화 직접 계산 부담 줄음
- **월정액으로 베이스 매출 + 추가 구매로 헤비 유저 매출 추가 확보**
- **글자수 차등으로 비용 정합성 유지** → 작은 분석 = 작은 차감, 큰 분석 = 큰 차감
- **옵션 추가 = +1 NAT** → 사용자가 "추가 가치 = 추가 비용" 직관 이해

## 3. Alternatives Considered (다른 후보들)

### ⚠️ 정직한 회고
본 결정은 **체계적 비교 없이 직감적으로 하이브리드를 채택**. 다음 대안들은 검토 깊이가 얕았음. v2 에서 보강 예정.

### 대안 A — 단순 월정액 무제한
- 예: 9,900원 / 월, 무제한 분석
- 장점: UX 간단, 가격 책정 단순
- 단점: **헤비 유저가 API 비용 폭주시킬 수 있음**. 1인 운영 = 위험

### 대안 B — 회차당 결제 (페이고)
- 예: 분석 1건 = 500원
- 장점: 사용량 정확히 비용 반영
- 단점: 매번 결제 = 사용 마찰 큼, 작은 카피 결제 부담

### 대안 C — 펙 결제 (선결제)
- 예: 10,000원 충전 = 분석 X건
- 장점: 펙 모델로 매출 안정
- 단점: 글자수 차등 어려움 (펙 = 균일가 가정)

### 대안 D — 하이브리드 (채택)
- 월정액 베이스 + 소진 시 추가 구매 + 글자수 차등
- 장점: 각 유저층 (라이트/미들/헤비) 모두 커버
- 단점: 모델 복잡도 증가, 사용자 학습 비용

## 4. EV Analysis (기댓값 분석, 소급 적용)

> 공식: **E = P(win) × Gain − P(loss) × Loss**

### 대안 A — 단순 월정액
```
P(win)   = 0.4
근거: UX 좋지만 헤비 유저가 손실 발생시키면 비즈니스 무너짐

Gain     = 가입 전환율 ↑, 단순 가격
Loss     = 헤비 유저 1명이 월 5만원어치 API 사용 = 적자

EV       ≈ 살짝 음수 (1인 운영 리스크 큼)
```

### 대안 B — 페이고
```
P(win)   = 0.5
근거: 비용 정합성 좋지만 사용 마찰로 가입 전환 ↓

Gain     = 비용 1:1 매핑, 안전
Loss     = 매번 결제 부담, 사용 빈도 ↓

EV       ≈ 0
```

### 대안 C — 펙 결제
```
P(win)   = 0.55
근거: 펙 매출 좋지만 글자수 차등 적용 어려움

Gain     = 매출 안정성
Loss     = 작은 분석/큰 분석 비용 차이 무시 → 일부 적자 가능

EV       ≈ 약한 양수
```

### 대안 D — 하이브리드 (채택) ⭐
```
P(win)   = 0.75
근거:
- AI API 비용이 글자수에 비례 → NAT 차등 소모로 정합성 확보
- 작가마다 사용 패턴 달라서 → 라이트(월정액)/헤비(추가구매) 양측 커버
- 심리적 경제학 → "NAT" 라는 추상 단위가 키워드 자유도 ↑

Gain     = 베이스 매출 + 헤비 매출 + 비용 정합 + UX 균형
Loss     = 모델 복잡도, 사용자 학습 비용

EV       ≈ 강한 양수
```

### 비교

| 옵션 | EV | P(win) |
|-----|----|--------|
| A 월정액 | ≈ 음수 | 0.4 |
| B 페이고 | ≈ 0 | 0.5 |
| C 펙 | 약한 양수 | 0.55 |
| **D 하이브리드** | **강한 양수** | **0.75** |

## 5. 채택 근거 (요약)

### 1. AI API 비용이 글자수에 비례
→ 균일 가격은 비용 정합성 깨짐. **NAT 차등 소모** 가 자연스러운 해법.

### 2. 작가마다 사용 패턴 다름
→ 단일 모델은 어느 한 층 (라이트/미들/헤비) 을 잃음.
→ **월정액 + 추가 구매** 가 모든 층 커버.

### 3. 심리적 경제학 (키워드 자유도)
→ 원화 직접 표시는 매번 비용 계산 부담.
→ "NAT" 추상 단위로 **결제 빈도 ↓ + 사용 자유도 ↑**.
→ 게임 업계의 다이아몬드/젬 모델과 동일 원리.

## 6. Consequences (결과 예상)

### Positive
- **매출 다층화** — 베이스 (월정액) + 추가 (헤비 유저)
- **비용 정합성** — 글자수 차등으로 적자 위험 ↓
- **확장성** — 옵션 추가 시 +1 NAT 룰로 단순 확장
- **포트폴리오 가치** — "AI 비용 구조 기반 과금 설계" 사례

### Negative
- **모델 복잡도** — 사용자가 NAT 차감 규칙 이해 필요
- **랜딩/온보딩 부담** — NAT 가 뭔지 설명해야 함
- **추가 구매 마찰** — 소진 시점에 결제 페이지 진입 = 일부 이탈

### Neutral
- 베타 단계 = 무료 운영 → 매출 검증은 패들 결제 오픈 후
- 글자수 구간 (3K/6K/10K) 은 추후 데이터 보고 조정 가능

## 7. Open Questions

- 일괄 분석 (10화 청크) 의 NAT 차감 규칙 정밀화 (현재는 회차별 합산)
- 무료 NAT (주 3개) 의 적정성 — 너무 많으면 결제 전환 ↓, 너무 적으면 가입 ↓
- 추가 구매 가격대 (4,900 / 8,900 / 15,900) 의 매력도 검증
- 글자수 구간 경계 (3K/6K/10K) 의 실제 분포 적합도
- 패들 도입 후 환불 정책의 NAT 처리 (이미 사용한 NAT 환불 가능 여부)

## 8. Revisit Criteria

아래 조건에서 이 결정을 재검토한다:

### 매출 장기 제한 ⚠️ (주요 트리거)
- 패들 결제 오픈 후 **3개월 내 월 매출 100만원** 미달 시 모델 자체 재검토
- 이 경우 가능한 방향:
  - 가격 인하
  - 무료 NAT 확대로 입소문
  - B2B 전환 (출판사/매니지먼트 제휴)

### 사용 패턴 데이터 누적 후
- 베타 종료 시점에 **실제 작가의 평균 NAT 소모 데이터** 수집
- 데이터가 가정과 다르면 (예: 평균 소모 < 50 NAT) → 가격/구간 조정

### 외부 충격
- AI API 가격 50% 이상 변동 시 (인상/인하 양방향)
- 경쟁사가 파괴적 가격 (월 무제한 5천원 등) 출시 시
- 한국 웹소설 시장 자체 축소 시

## Appendix A: NAT 모델의 사고 과정 (회고)

이 모델은 **체계적 비교 없이 직감으로 결정** 됐음. 이유:
- 베타 출시 압박 (시간 부족)
- 1인 운영 = 결정 권한 집중 = 토론 없음
- 게임 업계의 다이아몬드 모델 + SaaS 의 월정액 모델을 자연스럽게 결합

**돌이켜 보면 EV 분석 결과도 D 가 최선** 으로 나옴 → 직감의 신뢰도 검증된 셈.
다만 ADR 의 가치는 "결정의 정당화" 가 아니라 "결정 과정의 기록" 이므로,
**"비교 검토 부족"** 도 정직하게 §3 에 명시.

## Appendix B: NAT 라는 네이밍의 가치

원화 직접 표시 vs NAT 추상 단위:

| 항목 | 원화 표시 | NAT 추상 |
|-----|---------|---------|
| 결제 빈도 | 매번 | 충전 후 자유 |
| 비용 인지 | 매번 부담 | 충전 시 1회 |
| 사용 자유도 | 낮음 (계산 부담) | 높음 |
| 마케팅 | 어려움 (가격 직접 노출) | 쉬움 ("100 NAT 증정") |

→ 게임 업계가 검증한 패턴 (Riot Points, V-Bucks, 카카오 캐쉬) 의 SaaS 적용.
