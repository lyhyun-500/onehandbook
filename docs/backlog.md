## 🟡 P1 — 중요, 이번 스프린트 안에

### P1-001: OAuth 로컬 개발 리다이렉트 수정
- **발견**: 2026-04-21, 어드민 테스트 중
- **증상**: `localhost:3000` 에서 Google 로그인 시 `novelagent.kr` 로 랜딩됨
- **원인 추정**: Supabase `Authentication → URL Configuration → Redirect URLs` 에 localhost 미등록
- **임시 회피**: 네이버 로그인 사용
- **영향 범위**: LEE 만 (로컬 개발 환경) — 실제 유저 영향 없음
- **해결 방법**:
  1. Supabase 대시보드 → Authentication → URL Configuration
  2. Redirect URLs 에 추가:
     - `http://localhost:3000/**`
     - `http://localhost:3000/auth/callback`
  3. 저장 후 재테스트
- **예상 시간**: 5분

### P1-002: `/login` 후 원래 경로 복귀 로직
- **발견**: 2026-04-21, 어드민 작업 중 웹 Claude 지적
- **증상 가설**: `/admin` 접근 → 미인증 → `/login` 리다이렉트 → 로그인 후 `/admin` 이 아니라 `/` 또는 `/studio` 로 이동할 가능성
- **검증 필요**: 실제 `proxy.ts` 의 리다이렉트 로직이 `?next=` 같은 파라미터 지원하는지 확인
- **사용자 영향**: 어드민 작업 시 매번 URL 재입력 필요 → 운영 UX 저하
- **예상 시간**: 30분 ~ 1시간 (proxy.ts 수정 + 테스트)

### P2-002: 어드민 role UPDATE 자동화
- **발견**: 2026-04-21
- **증상**: admin 계정 지정할 때 SQL 수동 실행 필요
- **개선**: CLI 스크립트 또는 `/admin/settings` 에서 다른 유저를 admin 승격 가능하게
- **근거**: 나중에 CS 담당자 영입 시 유용
- **예상 시간**: 2~3시간 (다음 스프린트 이후)

### P2-003: 유저 통계에서 admin 계정 제외
- **발견**: 2026-04-21, 패턴 A (단일 계정) 채택 후
- **증상 가설**: DAU/MAU 등 지표 계산 시 admin 계정 포함되면 노이즈
- **해결**: 집계 쿼리에 `WHERE role = 'user'` 추가 습관화 또는 뷰 생성
  ```sql
  CREATE VIEW regular_users AS
  SELECT * FROM users WHERE role = 'user' AND deleted_at IS NULL;
  ```
- **예상 시간**: 30분
