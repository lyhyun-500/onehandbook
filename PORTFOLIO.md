# Novel Agent — 포트폴리오 요약

웹소설 작가를 위한 **AI 흥행 분석 SaaS**를 설계·구현한 프로젝트입니다. 취업용 포트폴리오·이력서·노션에 옮겨 쓸 수 있게 정리했습니다.

---

## 1. 한 줄 요약 (헤드라인)

**Novel Agent** — 웹소설 원고를 업로드하면 장르·플랫폼 맥락을 반영해 **흥행 가능성 분석 리포트**를 주는 **작가용 AI 스튜디오**(Next.js + Supabase + 멀티 LLM).  
개인 작품·회차 단위 분석, 크레딧(NAT), 비동기 작업·실시간 상태, 트렌드 RAG(Chroma) 등 **엔드투엔드 제품** 형태로 구현.

---

## 2. 포트폴리오 카드용 (3~5문장)

웹소설 작가를 위한 **AI 흥행 분석 SaaS**를 설계·구현했습니다. **Next.js(App Router)** 프론트와 **Supabase**(Auth, Postgres, RLS, Realtime)로 로그인·데이터·권한을 일원화하고, **회차/통합 분석 파이프라인**(배치·병합·진행률·취소)과 **NAT 크레딧**으로 과금·한도를 모델링했습니다. 분석은 **Anthropic / Google** 등 프로바이더 추상화로 연결하고, **장르 트렌드 RAG**는 **ChromaDB + 임베딩**으로 검색합니다. **Vercel**에 배포해 실서비스 URL로 데모 가능하며, 관측·안정성을 위해 **Sentry** 등을 고려한 구조입니다. 트렌드 리포트 **이중 저장(Supabase + Chroma)**, 학습용 **`training_logs`**, 헤더 **알림 패널 무한 스크롤** 등 운영·확장 요소를 반영했습니다.

---

## 3. 기술 스택 (이력서 표용)

| 영역 | 스택 |
|------|------|
| 프론트 | React 19, Next.js 16, TypeScript, Tailwind CSS |
| 백엔드·BaaS | Supabase (Auth, Postgres, RLS, Realtime 구독) |
| AI | Claude(Anthropic), Gemini(Google), 프롬프트·파이프라인 모듈화 |
| 데이터 / RAG | ChromaDB, `@chroma-core/default-embed`, 트렌드 청크·메타데이터 |
| 인프라 | Vercel 배포, 환경 변수로 멀티 환경(로컬 Chroma vs 원격 등) 분리 |
| 품질 | ESLint, TypeScript strict, Sentry 연동 준비 |

---

## 4. 구현 범위 (설계·개발 포인트)

**제품·도메인**

- 회차 단위 분석, **통합(다회차) 분석**(청크·병합·진행률·예상 시간·재시도).
- **NAT**로 분석 1회당 소모량·옵션 가산·잔액·빌링 안내 흐름.
- 휴대폰 인증·동의·소셜 로그인 등 온보딩·계정 정책과 DB 스키마 연동.

**아키텍처·품질**

- `analysis_jobs` **비동기 작업 + Realtime**으로 분석 상태 동기화, 낙관적 UI와 재동기화.
- Supabase **RLS**로 사용자별 데이터 격리.
- AI 호출 **프로바이더 추상화**, 분석 프로필·프롬프트 구성 분리.

**운영·확장**

- 헤더 **알림 패널**: 최근 7일·**커서 기반 페이지네이션**, 로딩/종료 문구.
- **트렌드 데이터**: `reports`(Supabase) + **원격 Chroma**에 임베딩 저장하는 **이중 인제스트** 플로우.
- **Ollama 파인튜닝 대비** `training_logs`(질문·답변 쌍, 큐레이터 `is_good` 표시, RPC로 권한 분리).

---

## 5. 이력서용 불릿

- Next.js·TypeScript 기반 **풀스택 웹앱** 설계·구현(인증, 결제 전 크레딧, 스튜디오, 분석 UI).
- Supabase로 **스키마·RLS·Realtime**을 적용해 다중 사용자·실시간 작업 상태를 안전하게 처리.
- **LLM 파이프라인**(회차/통합 분석, 배치, 에러·취소·진행률)을 구현해 **장시간 작업**을 제품 수준으로 관리.
- **벡터 DB(Chroma)** 와 임베딩으로 **장르 트렌드 RAG** 검색 경로 구축.
- **Vercel** 배포 및 환경 분리로 **데모 URL** 기반 포트폴리오 제시 가능.

---

## 6. 면접 스토리 (STAR용)

- **상황:** 원고 분석은 시간이 길고 실패·재시도가 잦음 → 단순 API 호출이 아니라 **작업 큐·상태머신**이 필요함.
- **과제:** 사용자에게 **진행률·취소·알림**을 주고, 서버와 **권한·과금**을 맞춰야 함.
- **행동:** `analysis_jobs` + Realtime, NAT 모델, 프로바이더 분리, RLS.
- **결과:** “실시간 피드백 + 크레딧 + 안전한 데이터 경계”가 있는 **B2C 크리에이터 툴** 형태로 설명 가능.

**보조 질문 대비**

- **왜 Supabase?** Auth·DB·Realtime을 한곳에서 쓰고 RLS로 멀티테넌시 처리.
- **왜 Chroma?** 트렌드 문서를 청크·임베딩해 **검색 품질**을 맞추기 위함(컬렉션·임베딩 일치).
- **스케일:** MVP는 개인 스튜디오, 루트 `README.md`에 **확장 로드맵**(탐색·체인 등)을 분리.
