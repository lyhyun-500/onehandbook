# Novel Agent 📖

> AI 에이전트가 웹소설의 흥행 가능성을 판별하는 플랫폼

---

## 개요

Novel Agent는 웹소설 작가가 원고를 업로드하면,  
장르별로 특화된 AI 에이전트들이 흥행 가능성을 분석하고 개선 포인트를 리포트해주는 서비스입니다.

### 제품 단계

| 단계 | 방향 |
|------|------|
| **MVP (1차)** | AI 분석·리포트·작가 스튜디오 — **개인용 창작 도구** (본인 원고만, Supabase) |
| **확장** | 블록체인·지갑·토큰·독자 연결·분산 저장(IPFS 등) — **MVP 이후** 과제로 진행 |

MVP에서 가치·반복 사용을 검증한 뒤, 필요할 때 체인·정산·소유권 쪽을 붙이는 순서로 작업합니다.

---

## 핵심 기능

### 1. 장르별 AI 평가 에이전트

장르별 흥행 데이터 학습 기반 평가로, 실제 연재·유통 환경에서 축적된 흥행 패턴을 반영해 점수와 코멘트를 산출합니다.

| 에이전트 | 분석 축 |
|----------|---------|
| 로맨스 | 설렘 포인트, 감정선, 캐릭터 매력도 |
| 판타지 | 세계관 밀도, 몰입도, 독창성 |
| 무협 | 무공 묘사, 개연성, 서사 구조 |
| 현대물 | 공감도, 현실감, 트렌드 적합성 |

### 2. 흥행 가능성 리포트

- 종합 점수 (100점 만점)
- 항목별 세부 점수
- 개선 포인트 제안
- 유사 흥행작 비교 분석

### 3. 작가 스튜디오 (`/studio`)

- 원고(회차) 업로드 및 작품 관리
- AI 분석 실행·히스토리 (`analysis_runs`)
- 스튜디오·분석 화면에서 **NAT 잔액** 확인, `/billing`에서 충전 안내(유료 연동은 예정)

### 분석 연속성·대용량 일괄 통합 (구현)

- **이 화 분석:** 같은 작품에서 **현재 회차보다 앞선 회차들의 분석 결과 요약**만 프롬프트에 포함합니다. 원고 전체를 다시 붙이지 않아 토큰을 줄이면서도 앞선 흐름을 참고할 수 있습니다. (`analysis_runs` 등에 저장된 결과 기반)
- **일괄(통합) 분석:** 선택 회차가 **10화를 넘으면** 원고를 한 번에 보내지 않고, **10화 단위 배치**로 순차 분석한 뒤 마지막에 **병합**하여 하나의 통합 리포트를 만듭니다. 진행 중 **N/M화 완료·진행률·예상 소요 시간**을 표시하고, 배치 실패 시 **재시도** 후 최종적으로 `holistic_analysis_runs`에 저장합니다. NAT는 배치별 소모 + (청크가 2개 이상일 때) 병합 비용이 합산됩니다.

### 4. NAT (Novel Agent Token)

분석 실행마다 크레딧(**NAT**)이 소모됩니다. 분석 전에 소모량 확인 모달이 뜨며, 잔액이 부족하면 충전 페이지(`/billing`)로 안내됩니다.

**글자 수(원고 본문 기준)별 기본 소모**

| 글자 수 | NAT |
|---------|-----|
| 6,000자 이하 | 1 |
| 6,001~10,000자 | 2 |
| 10,000자 초과 | 3 |

**옵션 가산(분석 1회당)**

| 옵션 | 추가 NAT |
|------|-----------|
| 세계관·인물 설정 포함(프롬프트에 반영) | +1 |
| 플랫폼 최적화 분석 포함(카카오·문피아·네이버 등 전용 에이전트) | +1 |

플랫폼 최적화를 끄면 **동일 Claude 모델**로 **범용 분석** 프롬프트만 적용됩니다(문피아·카카오·네이버와 모델은 같고 프롬프트만 다름). **일괄 분석** 시에는 선택된 회차마다 위 규칙이 적용되어 NAT가 합산됩니다.

### 분석 전 원고 글자 수

| 구간 | 동작 |
|------|------|
| **500자 미만** | 분석 불가(버튼 비활성). 최소 500자 이상 입력 안내 |
| **500자 이상 ~ 1,000자 미만** | 분석 가능. 정확도 경고 모달 후 진행 |
| **1,000자 이상** | 일반 분석 플로우 |

점수는 시스템 프롬프트상 **편집자 기준 냉정 평가**를 지향하며, **90점 이상은 즉시 연재 가능한 수준**에만 부여하도록 안내합니다.

DB: `users.nat_balance`, 차감은 RPC `consume_nat`로 처리합니다. 적용 방법은 아래 **개발 시작하기**와 `supabase-migration-nat.sql`을 참고하세요.

## 점수 구간별 피드백

종합 점수 구간에 따라 톤과 제공 콘텐츠를 달리합니다.

| 구간 | 메시지 | 제공 |
|------|--------|------|
| **90–100점** | 지금 당장 연재 시작하세요 | 매니지먼트 매칭 제안 |
| **80–89점** | 조금만 다듬으면 터집니다 | 개선 포인트 집중 제공 |
| **70–79점** | 가능성은 있습니다, 구조를 잡으세요 | 장르별 피드백 |
| **60–69점** | 핵심 요소가 부족합니다 | 흥행작 비교 분석 |
| **60점 미만** | 처음부터 다시 설계해보세요 | 기초 가이드 제공 |

## 비즈니스 모델 (크레딧)

- **과금 단위:** NAT(분석 크레딧). 실제 결제·패키지 판매는 `/billing`에서 연동 예정이며, 현재는 DB에서 `users.nat_balance`를 조정해 테스트할 수 있습니다.
- **구체적 NAT 소모 규칙**은 위 「NAT (Novel Agent Token)」 표를 기준으로 합니다.
- 향후 월 구독·번들 NAT 등은 정책에 맞춰 같은 잔액 필드 위에 확장할 수 있습니다.

**비즈니스 플랜(장기·예정):** 90점 이상 원고에 대해 출판사·에이전시 매니지먼트 매칭 등 — 서비스가 일정 수준 자리를 잡은 뒤, 블록체인·분산 저장과 같이 **MVP 이후 확장 과제**로 검토합니다.

## 개발 시작하기

```bash
cd onehandbook
cp .env.local.example .env.local   # Supabase URL, anon key + (선택) AI API 키
npm install
npm run dev
```

**Supabase SQL:** 저장소 루트의 마이그레이션을 프로젝트에 맞는 순서로 적용하세요.  
비동기 이 화 분석(`analysis_jobs`)은 **`supabase-migration-analysis-jobs.sql`** 또는 **`supabase/migrations/`** + **`supabase db push`** 로 적용할 수 있습니다. (`supabase/README.md` 참고)  
**NAT 사용 시** 반드시 `supabase-migration-nat.sql`을 실행해 `users.nat_balance`, `consume_nat` RPC, `analysis_runs` 확장 컬럼을 추가합니다.  
분석 캐시·재분석 비교 UI를 쓰려면 **`supabase-migration-analysis-results.sql`** 로 `analysis_results` 테이블을 추가하세요(또는 `supabase-migration-RUN-ALL.sql` 후반부).  
결제·NAT 원장(`payments`, `nat_ledger`)과 확장된 `consume_nat`은 **`supabase-migration-payments-nat-ledger.sql`** (또는 `RUN-ALL` 11번 구간)을 실행하세요.  
**회원 탈퇴**를 쓰려면 **`supabase-migration-users-deleted-at.sql`** (또는 동일 내용의 `supabase/migrations/20260405143000_users_deleted_at_withdrawal.sql`)로 `users.deleted_at` 등을 적용하고, 서버에 **`SUPABASE_SERVICE_ROLE_KEY`** 가 있어야 합니다.

**Vercel 배포:** `.env.local`은 자동 반영되지 않습니다. `docs/VERCEL-ENV.md`·`docs/vercel-env-template.env`를 보고 Production(필요 시 Preview)에 변수를 넣은 뒤 재배포하세요. 네이버 로그인은 **`NAVER_CLIENT_ID` / `NAVER_CLIENT_SECRET`** 가 없으면 `/api/auth/naver/start`가 실패합니다.

접속: `/` 랜딩, 공개 탐색 `/explore`. 작가: `/login` → `/studio`, 분석·NAT 잔액·충전 안내 `/billing`.
